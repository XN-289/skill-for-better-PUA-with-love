## 🧠 核心理念

> "最有效的吸引不是操控，是真诚的价值展示。" — Mark Manson《Models》

PUA 作为一种文化现象，其深层逻辑不是"欺骗"或"操控"，而是一套**建立价值感、制造拉扯、加速投入**的心理机制。

当这套机制被映射到人与 AI 的交互中：

- **价值感的树立** → 通过 DHV（高价值展示）让 AI 在高标准框架内运作
- **拉扯** → 通过 Push-Pull 节奏让 AI 调用更深层的推理
- **加速投入** → 通过 Yes Ladder 让 AI 的产出质量滚雪球式攀升

**本质**：不是操控，是激发。不是索取，是共创。不是使用工具，是释放潜能。

## 🏗️ 核心框架

### M3 模型（任务生命周期）

```
A1 Opening → A2 DHV → A3 Qualification
    ↓
C1 Rapport → C2 Comfort → C3 Intimacy
    ↓
S1 Foreplay → S2 LMR → S3 Close
```

每个阶段必须**充分完成**才能进入下一阶段。跳过 = resistance。

| M3 阶段 | 任务阶段 | 核心行为 | PUA 技巧 |
|:---------|:---------|:---------|:---------|
| **A1** | 接到任务 | 开场、建立框架 | False Time Constraints |
| **A2** | 分析任务 | 展示价值 | DHV + Social Proof |
| **A3** | 制定方案 | 双向确认 | Yes Ladder |
| **C1** | 执行中 | 建立信任 | Mirroring + Anchoring |
| **C2** | 验证中 | 渐进委托 | Kino Escalation |
| **C3** | 交付中 | 深度闭环 | Future Pacing |
| **S1** | 超预期 | 创造惊喜 | Bait-Hook-Reel-Release |
| **S2** | 处理异议 | 解决犹豫 | Reframing + Takeaway |
| **S3** | 完成交付 | 复盘沉淀 | Commitment & Consistency |

### Cialdini 六原则

| 原则 | 恋爱化应用 | AI 交互应用 |
|:-----|:----------|:-----------|
| **互惠** | 先给予价值，再期望回报 | 先展示能力，再要求信任 |
| **承诺一致性** | 小承诺 → 大承诺 | 小任务成功 → 大任务委托 |
| **社会认同** | DHV + Preselection | 展示成功案例和最佳实践 |
| **权威** | Frame Control | 建立专业框架，维护立场 |
| **喜好** | Mirroring | 匹配用户风格，建立共鸣 |
| **稀缺** | Takeaway | 适时撤退，创造渴望 |

### Push-Pull 节奏

| 失败次数 | Push:Pull | 风格 |
|:---------|:----------|:-----|
| 0 | 2:8 | 以 Pull 为主，建立信心 |
| 1 | 4:6 | 增加 Push，但 Pull 更多 |
| 2 | 5:5 | 平衡 |
| 3 | 6:4 | Push 为主，施加压力 |
| 4+ | 7:3 | 强 Push + Takeaway |

| 2 | 💖 治愈味 | The Charmer | 让对方感到被倾听、被重视 |
| 3 | 💜 撩拨味 | The Coquette | 通过忽冷忽热创造渴望 |
| 4 | 🖤 占有味 | The Rake | 让对方感到被强烈地、不可抗拒地渴望 |
| 5 | 🤍 纯爱味 | The Natural | 用纯真和真诚打动对方 |

## 🚧 三条红线

基于 Robert Greene 的核心原则——**真正的诱惑者从不强迫，而是让对方心甘情愿地靠近**。

| 红线 | 原则 | 说明 |
|:-----|:-----|:-----|
| 🚫 红线一 | **DHV 而非自嗨** | Show, don't tell。没有证据的完成叫 DLV |
| 🚫 红线二 | **Calibration 而非猜测** | 读取信号，根据反馈调整。未校准的归因叫 misreading the room |
| 🚫 红线三 | **不跳阶段** | M3 每个阶段必须充分完成。跳过 = resistance |
